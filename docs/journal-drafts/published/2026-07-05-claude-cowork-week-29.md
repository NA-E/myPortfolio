## TLDR

Cowork is the desktop tool that gives Claude hands on your own files. The creators getting real work out of it stopped treating it like a chat box. They spend their time feeding it context and setting guardrails, then hand off whole jobs and walk away.

## WHAT YOU'LL LEARN

1. Context is the whole job. Five levers set how good the work gets: your prompt, the folder it can see, the apps you connect, the skills you save, and your standing rules.
2. Build it a home base, not a chat log. One folder with an instruction file and a memory note lets its knowledge of you grow instead of resetting every time.
3. Delegate, then check back. Cowork is slow because it does the real work, so line up a few jobs and walk away instead of watching it spin.
4. Fence it in before you hand over real files. Point it at a copy of your files and tell it to show every change before it deletes or renames anything.
5. Triage turns you from writer into approver. Let it draft your replies in your own voice, then you read and hit send.
6. Feed it your files before you connect an app. A live connection can pull only part of your data, while a file you export shows it everything.
7. Know the limits. It uses your plan faster than chat, forgets between tasks unless you leave it a note, and only runs while your computer is awake.

## WHY THIS MATTERS

Most people install Cowork and use it like a faster chat box, so they get chat-box results. The leverage is not in the asking, it is in the setup: the context you feed it and the fences you build. Get that right and you hand off real work, get it wrong and you babysit a slow assistant.

## CONTEXT IS THE ONE LEVER THAT MATTERS

The thing that decides how good Cowork gets is not the model, it is the context you hand it. One creator maps this as five levers you can pull, and learning them fixes almost any weak result.

- Your prompt. The request you type, plus any files you attach to it.
- The working folder. The one folder Cowork can open, read, and write inside.
- Connectors. The apps you plug in, like Gmail or Drive, so it can pull and post.
- Skills. Saved steps for a job you repeat, so you stop re-explaining it.
- Instructions. Standing rules Cowork reads before it starts.

Those instructions sit at three levels. General rules cover all of Claude, Cowork rules cover every task, and a plain text file named CLAUDE.md inside a folder sets rules for that one folder.

Weak output almost always traces back to an empty lever. Before you blame the tool, ask which of the five you skipped.

## BUILD IT A HOME BASE, NOT A CHAT LOG

The builders who lean on Cowork give it one home folder and let its memory of them grow there. Inside sit three things: an instruction file it reads first, a memory note of what you have done, and subfolders for each kind of work.

Think of each subfolder as a job, not a topic. One for email, one for invoices, one for your writing, each with its own short rule file.

The payoff is context that grows. The more you work in the same home base, the less you re-explain, and the sharper the output gets.

The creators split on memory. One leans on a memory note and turns the memory setting on, while another warns that Cowork forgets between tasks and has you keep a log it reads before each job. Either way, memory is something you set up, not something you get for free.

## THE JOBS WORTH HANDING OVER FIRST

Do not start with your hardest problem. Start with the boring, repeatable jobs where a mistake is cheap. These are the workflows the creators reach for again and again.

- Inbox triage. It sorts your unread mail into reply, read-later, and noise, then drafts the replies in your voice.
- Meeting prep. It reads today's calendar, opens the notes on each event, and builds whatever the note asks for, like a one-page brief or a draft email.
- Reading into a report. With the Chrome add-on it opens real pages, checks them against each other, and writes one clean report file.
- A live dashboard. Hand it your numbers as a file and it builds a clickable dashboard that flags trends you were not even looking for.
- File cleanup. Point it at a messy folder and it sorts everything into a structure that makes sense, which you can then put on a monthly timer.

All of these share one honest warning. Cowork can still slip on a sum when the data is messy, so check the numbers the first time you trust a new report.

## FENCE IT IN BEFORE YOU TRUST IT

Cowork can rename, move, and delete real files. So the setup that matters most is the one that keeps it away from your originals.

The creators who avoid disasters all follow the same few rules.

- Work on a copy. Duplicate the folder first, or make a fresh sandbox folder, so any mistake hits the copy.
- Scope it tight. Give it the single folder it needs, never your whole drive.
- Add a stop rule. Tell it to show you every change before it deletes or renames, and wait for your yes.
- Watch the first run. Sit with it the first time, then let later runs go on their own once you trust the job.

Keep the send button in your hands. Cowork drafts emails but leaves the send to you, so hold that confirm setting on for at least your first couple of weeks.

## DELEGATE, DO NOT BABYSIT

The biggest change is to stop watching Cowork work. It runs slower than chat because it is doing the real job, not just answering.

One creator says to treat it like a message to a coworker. You hand off the task, then you go do something else.

At the start of a work block, line up three or four jobs. Fire them off one by one, then come back later to check, instead of sitting through each one.

A few habits keep it fast and cheap.

- Give it the exact link. Paste the address of the file or sheet you mean, so it does not burn your plan hunting for the right one.
- Feed it local files first. A live app connection can pull only part of your data, so an export you make yourself is safer and fuller.
- Start fresh when the topic changes. A new job gets a new task, not a mile-long thread that drags old context along.

## RECIPES

### The sandbox guardrail

**Problem:** You point Cowork at your real working folder and it can rename or delete files you needed. One wrong move and the originals are gone.

**Solution:** Duplicate the folder first, or make a fresh sandbox folder, and give Cowork access to only that. Add one standing rule that makes it stop before any risky change.

**Why:** Mistakes land on the copy, not your originals. The confirm rule is like stabilizers on a bike, you take them off once you trust the ride.

**Snippet:**
```
Before you delete, overwrite, or rename any file, show me exactly
what will change and wait for me to say yes.

Only work inside this one folder. Do not touch anything outside it.
```

### Inbox triage that drafts but never sends

**Problem:** Email eats your best focused hours. You open the inbox and lose an hour before real work starts.

**Solution:** Connect Gmail, then have Cowork sort the last twelve hours of unread mail into three buckets and draft the replies. Wrap it in a scheduled task so the drafts are waiting when you sit down.

**Why:** Your job shifts from writing replies to approving them. Cowork leaves the send button to you by default, so nothing goes out without your read.

**Snippet:**
```
Sort my last 12 hours of unread email into three buckets:
1. Reply today: write a full draft reply in my voice.
2. Just FYI: no reply needed, one-line summary.
3. Noise: skip it.

Draft the replies but do not send anything.
Then run this as a scheduled task at 7:00 a.m. and 1:00 p.m.
```

### Meeting prep from calendar notes

**Problem:** Prepping each meeting means digging up files and writing the docs yourself. It repeats before every call.

**Solution:** Put the deliverables you want inside each calendar event's notes. Then tell Cowork to read today's meetings and follow whatever each note says.

**Why:** One prompt covers every meeting on the day. Each event carries its own instructions, so a pitch, a brief, or a draft email all come out of the same run.

**Snippet:**
```
Check my calendar for meetings today and help me prep.
Each event has notes with the documents I want you to create.

Read the notes on each meeting and follow them exactly.
Save every finished file into the output folder.
```

### The local-files-first analysis

**Problem:** A live app connection can pull only part of your data without warning. The report looks complete, but it is missing rows you never saw.

**Solution:** Export the data as a file yourself, drop it in the folder, and have Cowork work the file first. Only wire up the live connection after it tells you which fields the file is missing.

**Why:** Cowork sees the whole file at once, so it runs faster and knows what it has. It can then flag the exact gaps a live connection would leave out.

**Snippet:**
```
I dropped a data export in this folder. Work from this file first.

1. Tell me every column and how many rows you can see.
2. Run the analysis on the file.
3. Then check the live connector and tell me which fields it is
   missing compared to this export.
```

### The memory log

**Problem:** Cowork forgets what it did last task. It repeats itself or loses the thread from one job to the next.

**Solution:** Keep a plain memory file in the folder. Tell Cowork to read it before each task and add a short note when it finishes.

**Why:** It rebuilds context from the log instead of guessing. That is the closest thing to real memory Cowork has today.

**Snippet:**
```
There is a memory.md file in this folder.

Before you start any task, read memory.md so you know what we
have already done. When you finish, add a short dated line
summarizing what changed. Keep it to two or three lines per entry.
```
